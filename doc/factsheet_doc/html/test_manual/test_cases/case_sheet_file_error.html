Case Factsheet File Errors¶

Purpose: demonstrate Factsheet response to file errors.

Setup¶

  1. Create a test data directory (for example, /home/Scratch/factsheet_test).

  2. In the test data directory create a file invalid.fsg containing a short test string (e.g., “No one expects the Spanish Inquisition.”

  3. In the test data directory create a directory no_backup.

  4. In directory no_backup create a factsheet inquisition.fsg with name “Inquisition” and title “No one expects”.

  5. Set user permissions on directory no_backup to read-only, (e.g. chmod u-w no_backup).

  6. If you use a file browser, set browser to show hidden files.

Steps – File Contents Not Factsheet¶

  1. Step: Start application with Helper Start application

    1. Expect: Application displays default factsheet.

  2. Step: Select Factsheet menu item File … > Open …. ( icon on far right of Factsheet identity line).

    1. Expect: Open dialog appears.

  3. Step: Change to test data directory.

    1. Expect: Open dialog shows directory contents.

  4. Step: Select invalid.fsg and click Open button.

    1. Expect: Open dialog disappears.

    2. Expect: Second window appears.

    3. New window may cover first window.

    4. Expect: Window title is “OPEN ERROR”.

    5. Expect: Factsheet name is “OPEN ERROR”.

    6. Expect: Factsheet title is “Factsheet not opened.”

    7. Expect: Factsheet summary includes the following, where local paths are replaced by ellipsis (…):

      Factsheet not open! could not read file.
Error source is UnpicklingError: could not find MARK.
Path: .../invalid.fsg

  5. Step: Check console for application “ERROR” log entries.

    1. Expect: Four “ERROR” entries:

      ERROR: Factsheet not open! could not read file.
ERROR: Traceback (most recent call last):
ERROR:    File ".../control_sheet.py", line nnn, in _model_from_path
    model = pickle.load(io_in)".
ERROR: "_pickle.UnpicklingError: could not find MARK"

  6. Step: Click OPEN ERROR window close icon (window title far right).

    1. Expect: Window disappears.

  7. Step: In Unnamed window, click window close icon .

    1. Expect: Window disappears.

    2. Expect: Application closes.

Steps – Save Failure¶

  1. Step: Start application with Helper Start application

    1. Expect: Application displays default factsheet.

  2. Step: Select Factsheet menu item File … > Open …. ( icon on far right of Factsheet identity line).

    1. Expect: Open dialog appears.

  3. Step: Change to no_backup in test data directory.

    1. Expect: Open dialog shows directory contents.

  4. Step: Select inquisition.fsg and click Open button.

    1. Expect: Open dialog disappears.

    2. Expect: Second window appears.

    3. New window may cover first window.

    4. Expect: Window title is “Inquisition”.

    5. Expect: Factsheet name is “Inquisition”.

    6. Expect: Factsheet title is “No one expects”.

    7. Expect: Factsheet summary is “Edit factsheet description here.”.

  5. Step: In the Inquisition window, Edit ( icon on left) factsheet title to “No one expects the Spanish Inquisition!”

    1. Expect: The text appears in field.

  6. Step: Select Factsheet menu item File … > Save ….

    1. Expect: Error dialog appears with the following message, where local paths are replaced by ellipsis (…).:

      Factsheet not saved! could not make backujp.
Error source is PermissionError: [Error 13] Permission denied:
'.../inquisition.fsg' -> '.../inquisition.fsg~'

    2. Expect: The following message in the console log, where local paths, line numbers, and pathlib errors are replaced (…, nnn, and <pathlib error>, respectively).:

      ERROR: Factsheet not saved! could not make backup.
ERROR: Traceback (most recent call last):
ERROR:   File ".../control_sheet.py", line nnn, in _open_file_save
    io_out = self._path.open(mode='xb')
ERROR:  <pathlib error>
ERROR:  <pathlib error>
ERROR: FileExistsError: [Errno 17] File exists: '.../inquisition.fsg'

    3. Expect: The following message in the console log, where local paths, line numbers, and pathlib errors are replaced (…, nnn, and <pathlib error>, respectively).:

      ERROR: During handling of the above exception, another exception
occurred:
ERROR: Traceback (most recent call last):
ERROR:   File ".../control_sheet.py", line nnn, in _open_file_save
    self._path.replace(str(self._path) + '~')
ERROR:  <pathlib error>
ERROR: PermissionError: [Errno 13] Permission denied:
'.../inquisition.fsg' -> '.../inquisition.fsg~'

    4. Expect: The following message in the console log, where local paths, line numbers, and pathlib errors are replaced (…, nnn, and <pathlib error>, respectively).:

      ERROR: The above exception was the direct cause of the following
exception:
ERROR: Traceback (most recent call last):
ERROR:   File ".../view_sheet.py", line nnn, in save_sheet
    self._control.save(p_path_update)
ERROR:   File ".../control_sheet.py", line nnn, in save
    with self._open_file_save() as io_out:
ERROR:   File ".../control_sheet.py", line nnn, in _open_file_save
    raise BackupFileError from err_replace
ERROR: factsheet.control.control_sheet.BackupFileError

  7. Step: In error dialog, click OK button.

    1. Expect: Error dialog disappears.

  8. Step: Click window close icon (window title far right).

    1. Expect: Data Loss Warning dialog appears.

  9. Step: Click Discard button

    1. Expect: Dialog disappears.

    2. Expect: Inquisition window disappears.

  10. Step: Click window close icon .

    1. Expect: Window disappears.

    2. Expect: Application closes.

Teardown¶

  1. Delete test data directory along with its contents.

  2. Check console for exceptions, GTK errors, and warning messages. There should be none other than ERROR log entries identified in steps above.
